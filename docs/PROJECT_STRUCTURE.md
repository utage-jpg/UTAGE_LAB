# PROJECT_STRUCTURE.md

UTAGE LAB の主要作業は、**新規診断の作成・記事の量産・ページ整理**です。
このファイルは、Claude Code がプロジェクト全体の役割を理解しやすくするための構造メモです。

---

# 1. このプロジェクトで多い作業

## A. 新規診断の作成
頻度が高い作業です。
想定する成果物
- 単体HTML診断ページ
- 必要なCSS / JS
- `index.html` からの導線追加

## B. 記事の量産
頻度が高い作業です。
想定する成果物
- 類型記事JSON
- 用語記事JSON
- 将来的に読み込みテンプレートと連動する構造

## C. ページ整理
頻度が高い作業です。
想定する成果物
- トップページ整理
- セクション順最適化
- スマホUI調整
- カードレイアウト整理

---

# 2. 基本ファイルの考え方

## index.html
メインポータルの中核。
以下の役割を持つ
- サイトの入口
- 診断導線
- 人気導線
- カテゴリ導線
- 課金導線
- 類型ポータルへの橋渡し

新規ツールや新規ページが増えた場合、基本的に `index.html` 側の導線更新もセットで考える。

## types/
記事データ置き場。
基本方針
- 記事は JSON 管理
- 類型ごとに1ファイル
- 命名規則を統一

---

# 3. 命名ルール

## 類型記事
- `types/iei.json`
- `types/ile.json`
- `types/ennea-1.json`
- `types/mbti-intp.json`

既存命名ルールがある場合はそちらを優先する。
なければ、小文字・ハイフン区切り・一貫性を優先する。

## 診断ツール
- 内容が分かる名前にする
- 例：`love-type.html`
- 例：`fortune-check.html`

# 4. 出力時の基本単位

## 診断系作業
最低限これを揃える
1. 本体ファイル
2. 保存先
3. `index.html` 更新

## 記事系作業
最低限これを揃える
1. JSONファイル
2. 保存先
3. 必要なら読み込み側更新

## 整理系作業
最低限これを揃える
1. 修正対象ファイル
2. レイアウト修正
3. スマホ確認前提の調整
