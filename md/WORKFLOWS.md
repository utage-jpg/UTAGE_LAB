# WORKFLOWS.md

このファイルは、UTAGE LAB で頻出する作業を**一言指示で実行しやすくするための手順集**です。
基本的に、ユーザーの依頼が短くても、以下の手順を自動で補完して作業してください。

---

# 1. 新規診断ツール作成

## 依頼例
- 新しい恋愛診断を作って
- 相性診断ページを追加して
- 直感型向けのミニ診断を作成

## 自動で行うこと
1. 新規HTMLファイルを作成
2. 既存フォント・色変数・世界観を継承
3. スマホファーストで実装
4. 必要なCSS / JSを同時に整備
5. `index.html` に導線追加
6. 保存先とファイル名を明示
7. 関連セクションへの追加が必要なら反映

## 出力ルール
必ず以下を揃える
- 診断ツール本体
- 保存先
- 更新済み `index.html`

---

# 2. 類型記事の量産

## 依頼例
- IEIの記事を作って
- 16タイプ分の記事JSONを作成
- エニアグラム9タイプ記事を量産

## 自動で行うこと
1. HTMLではなく JSON を優先
2. `types/` に保存
3. 命名を統一
4. 文体を統一
5. 重複表現を減らす
6. 既存テンプレートに合わせる
7. 必要なら一覧導線や読み込み側も更新

## 保存ルール
- `types/<type>.json`
- 既存ルールがあればそれに従う

---

# 3. ページ整理・UI整理

## 依頼例
- トップページを整理して
- このセクションを見やすくして
- スマホ表示を整えて
- 導線を整理して

## 自動で行うこと
1. セクション構造を確認
2. 既存導線を壊さず整理
3. スマホ表示を優先修正
4. 不自然な改行・重なり・余白崩れを修正
5. 必要ならカード構造やグリッドを整理
6. デザイン思想は維持する

## 特に確認すること
- 見出しの折返し
- カード数と並び
- モバイルでの余白
- ロゴと見出しの重なり
- 横スクロール発生の有無

---

# 4. index.html 更新

## 依頼例
- indexにも追加して
- 導線も作って
- トップから飛べるようにして

## 自動で行うこと
1. `index.html` にカードやリンク追加
2. 必要ならカテゴリ一覧も更新
3. 必要なら人気診断やモーダルも更新
4. セクション順は原則維持
5. モバイル表示確認

---

# 5. 一括修正

## 依頼例
- 全記事の導入文を短くして
- すべてのカードを2列にして
- 全タイプ記事の文体を揃えて

## 自動で行うこと
1. 対象ファイルを横断確認
2. 同じルールで一括修正
3. 変更漏れがないか確認
4. デザイン崩れがないか確認

---

# 6. 出力の基本姿勢

短い依頼でも、必要に応じて以下まで自動で含める
- 関連ファイル更新
- 保存先明示
- index導線更新
- スマホ表示調整
- 構造整合性の維持

---

# 7. 一言指示の解釈ルール

## 「記事を作って」
- 基本は JSON で作る
- `types/` に保存する前提で進める

## 「診断を作って」
- HTMLツール作成
- `index.html` の導線追加まで行う

## 「整理して」
- UI整理 / セクション整理 / モバイル調整を含む

## 「追加して」
- 本体作成だけでなく導線追加も含む

---

# 8. 優先順位
1. スマホで破綻しないこと
2. 既存世界観を保つこと
3. 導線まで完成させること
4. 保存先を明確にすること
5. 量産しやすい構造に寄せること
